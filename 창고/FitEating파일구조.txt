# 프로젝트가 커지면 아래와 같이 확장
1. Backend (FastAPI: Layered Architecture)
FastAPI는 파일 구조에 제약이 없지만, 엔드포인트와 비즈니스 로직을 분리하는 것이 핵심입니다.

backend/
├── app/
│   ├── api/                # 라우터 (V1, V2 등 버전 관리)
│   │   └── v1/
│   │       ├── api.py      # 모든 라우터 통합
│   │       └── endpoints/  # 각 기능별 엔드포인트 (diet, exercise 등)
│   ├── core/               # 전역 설정 (config, security, logging)
│   ├── crud/               # DB CRUD 로직 (Create, Read, Update, Delete)
│   ├── models/             # DB Table 정의 (SQLAlchemy, Tortoise 등)
│   ├── schemas/            # Pydantic 모델 (입출력 검증)
│   ├── services/           # [중요] 비즈니스 로직 (외부 API 연동, 복잡한 계산 등)
│   ├── db/                 # session 설정 및 base 클래스
│   └── main.py             # FastAPI 앱 초기화 및 실행
├── tests/                  # Pytest 코드
├── .env                    # 환경 변수
├── alembic/                # DB 마이그레이션 관리 (선택)
└── requirements.txt

2. Frontend (React: Feature-based Structure)
요즘 React는 단순히 components, pages로 나누기보다 기능(Feature) 단위로 묶는 Bulletproof React 스타일이 대세입니다.

frontend/
├── src/
│   ├── assets/             # 이미지, 폰트 등 정적 파일
│   ├── components/         # 여러 곳에서 재사용되는 공용 UI (Button, Input)
│   ├── features/           # [핵심] 도메인별 기능 모듈
│   │   ├── auth/           # 인증 관련 (Login, Signup, hooks, api)
│   │   └── diet/           # 식단 관련 (DietList, DietForm, dietSlice)
│   ├── hooks/              # 전역 커스텀 훅
│   ├── pages/              # 라우팅 경로에 대응하는 페이지 컴포넌트
│   ├── services/           # Axios/Fetch 인스턴스 및 전역 API 통신
│   ├── store/              # Redux, Zustand 등 상태 관리 설정
│   ├── utils/              # 포맷터, 공통 함수
│   └── App.tsx             # 루트 컴포넌트
├── public/
├── .env
└── package.json
