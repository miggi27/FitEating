1. 페이지 진입 시 자동으로 실행하기 (useEffect)
DietPage.jsx 파일 안에서 기존에 만든 getAiFeedback 함수 아래에 이 코드를 추가해 보세요.

import React, { useState, useEffect } from 'react'; // useEffect 추가
// ...
useEffect(() => {
  // 💡 컴포넌트가 마운트(렌더링)될 때 실행
  // 단, 영양소 데이터(totals)가 아직 0이거나 로딩 중일 때는 피하는 게 좋습니다.
  if (totals.kcal > 0) { 
    getAiFeedback(); 
  }
}, []); // []는 페이지가 처음 열릴 때 '딱 한 번' 실행하라는 의미

2. 자동 vs 수동 변동 주기 (전략)
모든 상황에서 자동이면 오히려 불편할 수 있죠(예: 아직 오늘 음식을 입력 안 했는데 AI가 잔소리부터 하는 경우). 이럴 땐 조건이나 상태값을 활용해야 합니다.

방법 A: 특정 데이터가 있을 때만 자동 실행
데이터가 아예 없는 빈 페이지일 때는 버튼을 눌러야만 실행되게 하고, 기록이 있을 때만 자동으로 부르는 식입니다.

useEffect(() => {
  // 칼로리가 500kcal 이상일 때만 자동으로 물어보기 (그 이하는 직접 버튼 클릭)
  if (totals.kcal > 500) {
    getAiFeedback();
  }
}, [totals.kcal]); // 칼로리 값이 바뀔 때마다 체크

방법 B: 사용자 설정(Settings)과 연동하기
나중에 Settings 페이지에 "AI 자동 피드백 활성화" 스위치를 만들고, 그 값을 가져와서 결정하는 방식입니다.

useEffect(() => {
  // settings.autoAiFeedback 같은 설정값이 true일 때만 실행
  if (settings.autoAiFeedback) {
    getAiFeedback();
  }
}, []);

3. 주의할 점: 무한 루프와 비용
무한 루프 방지: useEffect의 의존성 배열([])을 비워두면 페이지 진입 시 한 번만 실행되지만, 만약 [totals]를 넣고 getAiFeedback 내부에서 다시 totals를 건드리면 무한 호출이 일어날 수 있으니 주의하세요!

Ollama 부하: 자동 재생은 페이지를 들어갈 때마다 서버(맥미니)에 요청을 보냅니다. 테스트 단계에서는 버튼 클릭(수동) 방식을 유지하다가, 기능이 완성되면 특정 조건에서만 자동으로 전환하는 것을 추천합니다.

💡 추천하는 구성
처음에는 "오늘 먹은 음식이 1개 이상일 때"만 자동으로 한 번 호출되게 하고, 그 이후에 추가로 먹은 것에 대한 피드백은 새로고침 버튼을 눌러서 받게 하는 것이 가장 깔끔합니다.

이제 음식 페이지 들어가자마자 Gemma가 "오늘 식단은 이렇군요!"라고 반겨주면 진짜 완성된 서비스 느낌이 날 거예요!

useEffect(() => {
  // 1. 칼로리 데이터가 들어왔고
  // 2. 이미 피드백을 받은 상태가 아닐 때만 실행 (무한 루프 방지)
  if (summary?.total?.kcal > 0 && !aiFeedback) { 
    getAiFeedback(); 
  }
}, [summary, aiFeedback]); // summary나 aiFeedback이 변하면 다시 체크해라!
